# Security Policy

## Supported Versions

SerikaCord is under active development. **Only the latest version** receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < 1.0.5 | :x:                |

If you are running an older version, upgrade to the latest release before reporting a vulnerability.

## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

Please report vulnerabilities privately:

1. Email **security@serika.dev** with a detailed description and reproduction steps.
2. Alternatively, use [GitHub Private Security Advisories](https://github.com/serika-dev/SerikaCord/security/advisories/new).

Include the following in your report:

- Description of the vulnerability and its impact
- Steps to reproduce (proof of concept)
- Affected version / commit hash
- Suggested fix (if any)

### Response Timeline

- **Acknowledgement:** within 48 hours
- **Initial assessment:** within 5 business days
- **Fix or mitigation:** depends on severity, targeted for the next release

## Bug Bounty

**We do NOT offer a bug bounty program.** Security reports are voluntary and we do not provide monetary rewards or compensation for vulnerability reports.

## Disclosure Policy

- Please do not disclose the vulnerability publicly until a fix has been released.
- We will credit reporters in release notes unless you prefer to remain anonymous.
